Introducing Yeoman

Yeoman (n) 1. A noble servant, carrying out various duties for the sovereign. Associated with hard toil. 2. Your new favorite web application stack and development tool.

What is Yeoman?

site can be found here: yeoman.io

  • A command line tool for front-end development, that offers an improved tooling workflow in one easy CLI
    • Generate: Quickly get you started with scaffolding with popular MVC JS frameworks, Twitter Bootstrap, and HTML5 Boilerplate
    • Manage: Manage, install, update project dependencies
    • Test: Built in JS testing frameworks, run by PhantomJS (or Testacular)
    • Serve: Built in local HTTP server with Livereload capabilities for fast development
    • Build: Compresses images, concatenates and minifies scripts, and stylesheets, recompiles SASS/CoffeeScript, lints JS, uses revision names for files for a final build of your app

For installation, and requirements: Installing Yeoman Note: Yeoman requires Node 0.8.x currently.

Yeoman Can Generate Projects For You

built on top of Grunt.js

  • Generator templates allow you to scaffold out a project using a custom setup of boilerplates, frameworks and dependencies.
  • A robust API, to create your own generators
    •             yeoman init quickstart # Skip our questions and get a H5BP, jQuery and Modernizr base
            # And many more...Built in project generators:
            #Angular.js, Backbone.js, Ember.js, Mocha, Testacular
  • A robust API, to create your own generators
  • Demo time

Yeoman Can Manage Dependencies For You

with Bower from Twitter

  • Bower is a package manager for the web. Bower lets you easily install assets such as images, CSS and JavaScript, and manages dependencies for you.
  • In Bower, dependencies are listed in a ‘component.json’ file, similar to Node’s package.json or the Ruby Gemfile. This is useful for locking down the dependencies a project has.

  • Accessing Bower in Yeoman:

                    yeoman search  #search for packages
                yeoman install  #install a package by name
                yeoman update  #update packages in your project to lastest versions
  • Demo time

Yeoman Can Test For You

with Mocha (or Jasmine), using PhantomJS

  • By default Yeoman runs a Mocha test harness in a headless instance of PhantomJS.
  • When you generate a new project using yeoman init, yeoman also scaffolds out a basic set of Mocha unit tests that you can continue using to test your application.

  • Accessing PhantomJS in Yeoman:

                    yeoman test # run your tests
                yeoman server # will watch for changes, and run your tests for you!
                yeoman init --test-framework jasmine #Start with Jasmine instead of Mocha
  • Demo time

Yeoman Can Serve For You

using a built in Node.js server locally

  • Server on port 3501 that allows you to access a running version of your application locally
  • Also changes to any of the applications files cause the browser to refresh via LiveReload
  • Any changes to CoffeeScript or Compass files result in them being recompiled

  • Accessing Server in Yeoman:

                  yeoman server # serve all the things!
              yeoman server:dist # serves an optimized final version of app
              yeoman server:test # serves up the test suite and your app
              yeoman server:reload # forces server to use standard Livereload port: 35729
  • Demo time

Yeoman Can Build For You

also built with Grunt.js, that compresses, optimizes and tests.

  • Lints all JavaScript files against JSHint
  • Recompiles all CoffeeScript and SASS files for production
  • Uses r.js to compile and optimize any AMD modules
  • Concatenates and minifies all scripts and stylesheets
  • Compresses images using OptiPNG for PNG files and JPEGtran-turbo for JPEGs
  • Runs unit tests written against a headless WebKit browser (via PhantomJS)
  • Creates an Application Cache manifest via Confess.js

  • Accessing Build in Yeoman:

                yeoman build:dist # builds an optimized final version of app
            yeoman server:dist # serves an optimized final version of app
  • Demo time
